I read through GAME_[[DESIGN.md](http://DESIGN.md)](http://DESIGN.md) and UI_[[VIEWS.md](http://VIEWS.md)](http://VIEWS.md). Overall it's a strong directional doc — pillars, tone, stages, and view inventory all hang together. The biggest weakness is that it stays at the "what" level and rarely commits to numbers, content, or interaction detail, which is exactly where playability and mobile bets get made. Here's what stood out.

Playability gaps
No numbers anywhere. Tick rates, action durations ("minutes to hours" is a 60× range), Earth prices, ore→metal ratios, storage caps, fuel costs, survey time per asteroid. Without at least one column of placeholder values, "AFK-friendly but not punishing" is unfalsifiable.
No first-time user experience. Acceptance criteria say "understand the first loop in 5 minutes" but there's no scripted onboarding, no tutorial moments, no "what does the player see at t=0." Stage 3 jumps straight into the loop.
No end state or replay loop. Stage 6 is "system-scale corporation" then the doc stops. Is there a win screen? Prestige? New game+? Open Questions hints at this but it's load-bearing for retention.
AFK return ritual is missing. No offline-summary modal, no "while you were away" reward feedback. The single most important moment in an idle game.
Failure modes are vague. "Evacuation or suspension" is mentioned once for colonies; nothing for fleet (ships lost? stranded? bankruptcy?). Events are recoverable by fiat.
Tension between "active survey" and "AFK-friendly". If survey is the signature active mechanic, what does a player who only checks in twice a day actually do? Survey automation is mentioned but the manual/auto handoff isn't defined.
Click depth and one-tap fixes. "Resolve alert in one step" is asserted but not storyboarded. What's the actual button path from "oxygen low" alert to fixed?
No explicit non-goals. Combat, piracy, PvP, trading, crew morale events — saying "the game does NOT include X" prevents scope creep later.
Content gaps
No master resource list. Resources appear ad-hoc across mocks (Aluminum Panels, Pressure Valves, Comfort Goods, Habitat Glass, Electronics, Silicates…) with no canonical table, relationships, or recipe count target.
No ship catalog. Three cargo classes is a taxonomy, not content. How many ship hulls at launch? Stats? Tier curve?
No celestial body inventory. "System-scale" is undefined — Earth/Moon/NEAs only, or Mars, belt, Jovian moons? This drives art, map scale, and route math.
Research/automation tree is empty. Stages mention unlocks but no node count or branch shape.
Colony tiers undefined past tier 2. Open Questions even flags this. The whole "colonies create new material demand" pillar rests on it.
Events are 6 examples. Need a target count and frequency budget so they don't all collide or vanish.
No narrative/flavor layer. "Optimistic hard sci-fi" is a vibe, not content. No company name, no Earth-side context, no NPC voice for alerts/contracts.
Mobile gaps
Desktop→mobile mapping is missing. Desktop has 10 nav items, mobile has 5 ("More" is undefined). Which sections collapse where?
Right-inspector content has no mobile home. Desktop relies heavily on a contextual right panel; on mobile it becomes "bottom sheet" but there's no rule for which selections push a sheet vs. a full screen.
Map interaction model is unspecified for touch. Pinch zoom, pan vs select disambiguation, long-press, edge gestures. The map is the emotional centerpiece — this matters.
Touch targets, thumb zones, landscape/tablet not addressed. The mock has dense lists that may not pass 44px targets.
No PWA / installability / push notification story. AFK games live or die on "your colony needs oxygen" notifications.
Performance budget missing. Three.js focus views on a mid-range Android need a fallback. Stage 0 picks Three.js without acknowledging this.
Backgrounded app / battery / offline simulation. Mobile browsers throttle JS aggressively; the deterministic catch-up rules need a mobile-specific clause.
Coherence/structural notes
UI_[[VIEWS.md](http://VIEWS.md)](http://VIEWS.md) and GAME_[[DESIGN.md](http://DESIGN.md)](http://DESIGN.md) overlap and drift. Desktop nav is listed three times (Stage 1, North Star, UI_VIEWS) with slightly different orderings and item names (e.g., "Ops" vs "Command", "Milestones" vs "Objectives"). Pick one and have the others reference it.
Stage 0 → Stage 1 handoff is soft. The "Decision Gate" lists choices but no record of which choice was made. If those defaults are now decisions, promote them out of Stage 0.
Acceptance criteria mix gameplay feel with implementation. Splitting into "design acceptance" vs "build acceptance per stage" would make slices testable.
Open Questions is short for a doc this size. Pricing, tutorial, prestige, mobile parity rules, content counts all belong there.
What I'd add next, in priority order
A one-page numbers sheet (placeholder economy, tick model, time budgets per action).
A resource + recipe table and a ship + colony tier table — even rough.
Mobile interaction spec (nav mapping, sheet rules, map gestures, notification plan).
A first-15-minutes script of what a new player sees and does.
An AFK-return spec (summary modal contents, capping rules, "what stopped" reasons).

Those answers tighten the design meaningfully. A few observations on what they imply, what they resolve, and what new gaps they open.

Anno / Paragon Pioneers reframes the whole doc. This isn't really an idle game with a space skin — it's an incremental tier-gated production-chain builder where the active loop is layout and chain design and the idle loop is throughput. That's a different center of gravity than what the design doc currently implies. The "active survey" mechanic looks less load-bearing under this framing; the real signature mechanic is tier transitions — Earth demands metals → unlock Moon → Moon population demands habitats → unlock NEA mining → Mars unlocks → Belt unlocks. The doc should commit to this. The current doc reads more like "EVE-lite + idle"; the answer says "Anno-in-space + idle".

Prestige-with-sandbox-option is a clean answer to the end-state gap. Worth pinning down what carries over (tech? tier unlocks? credits? recipes?) and what the prestige currency is named/derived from. Universal Paperclips' "trust" or Anno's "investor coins" are good shapes.

True dual-target + hybrid sessions is the most consequential answer. It means there's no "desktop UI" and "mobile UI" — there's one interaction language that scales. The desktop right-inspector pattern in UI_VIEWS likely has to die (or become a bottom-sheet on both). It also means cloud save isn't optional, and notifications are mandatory because long check-ins on mobile depend on them.

New gaps these answers create:

Tier ladder content target. How many tiers, what's the curve, what's the last-tier gate? This is now the #1 content question.
Recipe count target. Anno launched with ~40 goods. Paragon Pioneers with ~25. Pick a number.
Prestige economy. What's preserved, what's the currency, what's the multiplier curve.
Cross-device save / auth. Implied but not in the doc.
Notification taxonomy. Which alerts push vs. in-app only, and what's the spam budget.
What gets resolved or downgraded:

AFK return ritual still matters but is now one of several check-in patterns rather than the main one.
"Survey" question: setup is manual, ops auto-continues. The earlier active/AFK tension dissolves.
Combat/PvP non-goals are easier to commit to — Anno-likes don't need them.
Want me to draft the tier ladder + recipe table next, since that's now load-bearing for everything else? Or take a pass at the dual-target interaction spec first?
